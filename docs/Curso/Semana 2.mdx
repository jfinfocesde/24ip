# 2. Introducción al HTML

## HTML5

### a) ¿Qué es HTML y para qué se usa?

HTML significa "HyperText Markup Language" (Lenguaje de Marcado de Hipertexto). Es el lenguaje estándar utilizado para crear y diseñar páginas web. HTML no es un lenguaje de programación, sino un lenguaje de marcado que define la estructura y el contenido de una página web.

Usos principales:

-   Crear la estructura básica de páginas web
-   Organizar el contenido en elementos como encabezados, párrafos, listas, etc.
-   Insertar imágenes, videos y otros medios
-   Crear enlaces a otras páginas o recursos
-   Construir formularios para recopilar información del usuario

### b) Historia y evolución del HTML

-   1989: Tim Berners-Lee inventa la World Wide Web
-   1991: Se publica el primer documento formal sobre HTML
-   1995: HTML 2.0 se establece como el primer estándar oficial
-   1997: HTML 3.2 introduce tablas y applets
-   1999: HTML 4.01 mejora la accesibilidad y los scripts
-   2000-2014: Desarrollo de XHTML como una versión más estricta de HTML
-   2014: HTML5 se convierte en la recomendación oficial del W3C, introduciendo nuevos elementos semánticos, soporte multimedia mejorado y APIs para aplicaciones web complejas

### c) Estructura básica de un documento HTML

Un documento HTML básico consta de los siguientes elementos:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título de la página</title>
</head>
<body>
    <h1>Encabezado principal</h1>
    <p>Este es un párrafo de ejemplo.</p>
</body>
</html>
```

Explicación de la estructura:

-   `<!DOCTYPE html>`: Declara que este es un documento HTML5
-   `<html>`: El elemento raíz que contiene todo el documento HTML
-   `<head>`: Contiene metadatos e información sobre el documento
-   `<meta charset="UTF-8">`: Especifica la codificación de caracteres
-   `<meta name="viewport">`: Ayuda con la responsividad en dispositivos móviles
-   `<title>`: Define el título de la página que aparece en la pestaña del navegador
-   `<body>`: Contiene todo el contenido visible de la página web

## Estructura básica de una página web

La estructura básica de una página web HTML consta de varios elementos fundamentales que definen el documento y su contenido. Aquí está una explicación de cada parte, seguida de un ejemplo:

1.  Declaración DOCTYPE:
    -   Informa al navegador que este es un documento HTML5.
    -   Siempre va al principio del documento.
2.  Elemento `<html>`:
    -   Es el elemento raíz que engloba todo el contenido HTML.
    -   El atributo 'lang' especifica el idioma del documento.
3.  Elemento `<head>`:
    -   Contiene metadatos sobre el documento.
    -   No se muestra directamente en la página.
4.  Elemento `<meta>`:
    -   Proporciona metadatos sobre el documento HTML.
    -   Comúnmente se usa para especificar la codificación de caracteres y la configuración de la ventana gráfica.
5.  Elemento `<title>`:
    -   Define el título de la página que se muestra en la pestaña del navegador.
6.  Elemento `<body>`:
    -   Contiene todo el contenido visible de la página web.


```html
<!DOCTYPE html>
<html lang="es">
<head>
    <!-- Especifica la codificación de caracteres del documento -->
    <meta charset="UTF-8">
    
    <!-- Configura la ventana gráfica para dispositivos móviles -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    
    <!-- Título de la página que aparece en la pestaña del navegador -->
    <title>Mi Primera Página Web</title>
</head>
<body>
    <!-- Encabezado principal de la página -->
    <h1>Bienvenido a Mi Primera Página Web</h1>
    
    <!-- Párrafo de contenido -->
    <p>Esta es una página web básica creada con HTML.</p>
    
    <!-- Enlace a otra página -->
    <a href="https://www.ejemplo.com">Visita Ejemplo.com</a>
</body>
</html>
```

Explicación adicional:

-   La estructura comienza con `<!DOCTYPE html>`, que declara que este es un documento HTML5.
-   El elemento `<html>` envuelve todo el contenido y el atributo `lang="es"` especifica que el contenido está en español.
-   Dentro de `<head>`, los elementos `<meta>` proporcionan información importante para el navegador:
    -   `charset="UTF-8"` especifica la codificación de caracteres.
    -   El `viewport` meta tag es crucial para el diseño responsivo en dispositivos móviles.
-   El `<title>` define lo que se muestra en la pestaña del navegador.
-   Todo el contenido visible va dentro del `<body>`.
-   En este ejemplo, el `<body>` contiene un encabezado `<h1>`, un párrafo `<p>`, y un enlace `<a>`.


## Etiquetas meta

Las etiquetas meta son una parte importante del elemento `<head>` en un documento HTML. Proporcionan metadatos sobre el documento HTML, que no se muestran en la página pero son utilizados por navegadores, motores de búsqueda y otros servicios web. Vamos a explorar las etiquetas meta más comunes y su uso.

### Codificación de caracteres

```html
<meta charset="UTF-8">
```
Define la codificación de caracteres para el documento HTML. UTF-8 es el estándar recomendado.

### Viewport

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```
Controla cómo se muestra la página en dispositivos móviles. Es crucial para el diseño responsivo.


### Descripción

```html
<meta name="description" content="Una breve descripción de tu página web">
```
Proporciona una descripción corta de la página, que los motores de búsqueda pueden mostrar en los resultados de búsqueda.

### Palabras clave (menos relevante actualmente)

```html
<meta name="keywords" content="HTML, CSS, JavaScript">
```
Aunque menos utilizada hoy en día por los motores de búsqueda, puede listar palabras clave relevantes para la página.

### Autor

```html
<meta name="author" content="Nombre del Autor">
```
Especifica el autor del documento.

Ejemplo completo:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="description" content="Una página de ejemplo mostrando diferentes etiquetas meta">
    <meta name="keywords" content="HTML, meta tags, SEO">
    <meta name="author" content="Tu Nombre">
    
    <title>Ejemplo de Meta Tags</title>
</head>
<body>
    <h1>Contenido de la página</h1>
</body>
</html>
```

## Comentarios en HTML

Los comentarios son una parte importante del código HTML que permite a los desarrolladores incluir notas o explicaciones dentro del código fuente sin que estos afecten la visualización de la página web.

Características principales de los comentarios en HTML:

1.  No se muestran en el navegador
2.  Son útiles para documentar el código
3.  Pueden ayudar en la depuración
4.  Pueden usarse para "comentar" temporalmente partes del código HTML

Sintaxis de los comentarios en HTML:

Los comentarios en HTML comienzan con `<!--` y terminan con `-->`

```html
<!-- Esto es un comentario en HTML -->
```

Ejemplos de uso de comentarios:

1. Comentario de una sola línea:

```html
<!-- Este es un encabezado principal -->
<h1>Bienvenidos a mi sitio web</h1>
```
2. Comentario de múltiples líneas:

```html
<!-- 
  Este es un comentario
  que abarca
  múltiples líneas
-->
```

Es importante tener en cuenta algunas mejores prácticas al usar comentarios:

1.  Sé conciso pero claro en tus comentarios.
2.  Actualiza los comentarios cuando actualices el código.
3.  No incluyas información sensible en los comentarios.
4.  No abuses de los comentarios; el código bien escrito a menudo se explica por sí mismo.

-   Google Chrome
-   Mozilla Firefox

1.  Chrome DevTools (en Google Chrome):

## Elementos y etiquetas HTML básicas

### a) Estructura de una etiqueta HTML

Una etiqueta HTML típica consta de:

<div className="flex justify-center m-12">
<img src="https://i.postimg.cc/k4s9SqbT/Anatom-a-de-Etiqueta.png" alt="" className="rounded-lg " />
</div>

-   Una etiqueta de apertura: `<nombre_etiqueta>`
-   Contenido
-   Una etiqueta de cierre: `</nombre_etiqueta>`

```html
<p>Este es un párrafo.</p>
```
Algunas etiquetas son "auto-cerrantes" y no requieren una etiqueta de cierre separada:

```html
<img src="imagen.jpg" alt="Descripción de la imagen">
```

Ejemplos de etiquetas autocerradas comunes:

-   **`<br>`:** Inserta un salto de línea.
-   **`<hr>`:** Inserta una línea horizontal.
-   **`<img>`:** Inserta una imagen.
-   **`<input>`:** Crea un campo de entrada para formularios.
-   **`<meta>`:** Proporciona metadatos sobre el documento HTML.
-   **`<link>`:** Enlaza a hojas de estilo, scripts externos, etc.


### b) La etiqueta `<div>` en HTML 

La etiqueta `<div>` en HTML es una de las más utilizadas. Significa "división" y se utiliza para agrupar bloques de contenido sin un significado semántico específico. Algunas características de la etiqueta `<div>`:

-   Es un contenedor genérico y no tiene ningún significado semántico por sí solo. Sólo indica una división lógica del contenido.
-   Se utiliza cuando no hay otra etiqueta semántica más apropiada como `<header>`, `<footer>`, `<article>`, etc.
-   Las divisiones `<div>` se pueden anidar y agrupar en bloques lógicos.
-   Se puede asignar clases o IDs a los `<div>` para aplicar estilos o manipular el bloque con JavaScript.
-   Sirve para maquetar y organizar el diseño mediante CSS, aunque no indique ningún tipo de contenido.
-   Ejemplos: `<div class="cabecera">`, `<div id="sidebar">`, `<div class="entrada">`
-   Se recomienda usar otras etiquetas semánticas como `<header>` o `<section>` cuando describan una región con sentido.
-   En HTML5 ya no es necesario agrupar toda la página dentro de un único `<div>`, a diferencia de versiones anteriores.

```html
<div>
  <h1>Mi primera sección</h1>
  <p>Este es un párrafo dentro de la sección.</p>
</div>
```

### c) Etiquetas de encabezado (`<h1>` a `<h6>`)
HTML proporciona seis niveles de encabezados:

```html
<h1>Encabezado de nivel 1 (el más importante)</h1>
<h2>Encabezado de nivel 2</h2>
<h3>Encabezado de nivel 3</h3>
<h4>Encabezado de nivel 4</h4>
<h5>Encabezado de nivel 5</h5>
<h6>Encabezado de nivel 6 (el menos importante)</h6>
```

Es importante usar los encabezados de manera jerárquica para una mejor estructura y SEO.

:::info
SEO son las siglas de Search Engine Optimization, que en español significa Optimización para Motores de Búsqueda. En pocas palabras, es el conjunto de técnicas y estrategias que se utilizan para mejorar la visibilidad de un sitio web en los resultados de búsqueda de motores como Google, Bing o Yahoo.
:::

### d) Párrafos (`<p>`) y saltos de línea (`<br>`)
Párrafos:

```html
<p>Este es un párrafo. Puede contener múltiples oraciones.</p>
<p>Este es otro párrafo separado.</p>
```
Saltos de línea:

```html
<p>Esta es la primera línea.<br>
Esta es la segunda línea en el mismo párrafo.</p>
```

### e) Formateo de texto (negrita, cursiva, subrayado)

Negrita:

```html
<strong>Este texto es importante</strong>
<b>Este texto también está en negrita</b>
```

Cursiva:

```html
<em>Este texto está enfatizado</em>
<i>Este texto está en cursiva</i>
```

Subrayado:

```html
<u>Este texto está subrayado</u>
```

:::tip
Nota: Se prefiere el uso de `<strong>` y `<em>` sobre `<b>` e `<i>` respectivamente, ya que proporcionan significado semántico además del formato visual.
:::

Otros elementos de formateo útiles:

`<mark>`: Para resaltar texto

```html
<p>Este es un <mark>texto resaltado</mark>.</p>
```

`<small>`: Para texto más pequeño

```html
<p>Este es un texto normal. <small>Este es un texto más pequeño.</small></p>
```

`<del>`: Para texto tachado

```html
<p>Este es un <del>texto tachado</del>.</p>
```

`<ins>`: Para texto insertado

```html
<p>Este es un <ins>texto insertado</ins>.</p>
```

`<sub>` y `<sup>`: Para texto insertado

```html
sub> y <sup>: Para subíndices y superíndices
```

### f) Listas en HTML 

Las listas en HTML se utilizan para agrupar y organizar información relacionada. Hay tres tipos principales de listas en HTML:

1.  Listas no ordenadas (Unordered Lists)
2.  Listas ordenadas (Ordered Lists)
3.  Listas de definición (Definition Lists)

Vamos a explorar cada una de ellas:

1.  Listas no ordenadas (Unordered Lists):

Las listas no ordenadas se utilizan cuando el orden de los elementos no es importante. Se crean con la etiqueta `<ul>` y cada elemento de la lista se define con la etiqueta `<li>`

```html
<ul>
    <li>Manzanas</li>
    <li>Plátanos</li>
    <li>Naranjas</li>
</ul>
```

2.  Listas ordenadas (Ordered Lists):

Las listas ordenadas se utilizan cuando el orden de los elementos es importante. Se crean con la etiqueta `<ol>` y cada elemento de la lista se define con la etiqueta `<li>`

```html
<ol>
    <li>Precalentar el horno</li>
    <li>Mezclar los ingredientes</li>
    <li>Verter la mezcla en el molde</li>
    <li>Hornear por 30 minutos</li>
</ol>
```

3. Atributos para listas ordenadas:

-   `type`: Cambia el tipo de numeración (1, A, a, I, i)
-   `start`: Cambia el número inicial de la lista

```html
<ol type="A" start="3">
    <li>Tercer elemento</li>
    <li>Cuarto elemento</li>
</ol>
```

4. Listas de definición (Definition Lists):

Las listas de definición se utilizan para mostrar términos y sus definiciones. Se crean con la etiqueta `<dl>`. Cada término se define con `<dt>` y cada definición con `<dd>`.

```html
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language, el lenguaje estándar para crear páginas web.</dd>
    
    <dt>CSS</dt>
    <dd>Cascading Style Sheets, usado para describir la presentación de un documento HTML.</dd>
</dl>
```
5. Listas anidadas: 

Puedes anidar listas dentro de otras listas.

```html
<ul>
    <li>Frutas
        <ul>
            <li>Manzanas</li>
            <li>Plátanos</li>
        </ul>
    </li>
    <li>Verduras
        <ul>
            <li>Zanahorias</li>
            <li>Brócoli</li>
        </ul>
    </li>
</ul>
```

### g) La etiqueta `<a>` en HTML: Creando enlaces

La etiqueta `<a>` en HTML es fundamental para crear hipervínculos, es decir, enlaces que te permiten navegar de una página a otra dentro de tu sitio web o a cualquier otra página en internet.

```html
<a href="url">Texto del enlace</a>
```

:::info
 Una URL (siglas en inglés de Uniform Resource Locator, o Localizador Uniforme de Recursos en español) es como la dirección de una casa en internet. Es la cadena de texto que identifica de manera única la ubicación de un recurso en la web, como una página web, una imagen, un video o cualquier otro contenido en línea.
:::

### h) Etiquetas Semánticas en HTML5

Las etiquetas semánticas son elementos HTML que transmiten significado sobre la estructura y el contenido de una página web, más allá de solo definir su apariencia. Estas etiquetas ayudan a los navegadores, motores de búsqueda y tecnologías de asistencia a entender mejor el contenido de la página.
Principales etiquetas semánticas y sus usos:

`<header>`
- Representa el encabezado de una sección o de la página.
- Típicamente contiene elementos introductorios o de navegación.

```html
<header>
  <h1>Título del Sitio</h1>
  <nav>
    <ul>
      <li><a href="#inicio">Inicio</a></li>
      <li><a href="#acerca">Acerca de</a></li>
    </ul>
  </nav>
</header>
```

`<nav>`
- Define una sección de navegación principal.
- Se usa para menús de navegación, índices o tablas de contenido.

`<main>`
- Engloba el contenido principal de la página.
- Debe ser único en el documento.

```html
<main>
  <h2>Contenido Principal</h2>
  <p>Aquí va el contenido principal de la página.</p>
</main>
```

`<article>`
- Representa un contenido independiente y autónomo.
- Ideal para entradas de blog, noticias o comentarios.

```html
<article>
  <h2>Título del Artículo</h2>
  <p>Contenido del artículo...</p>
</article>
```

`<section>`
- Define una sección temática del documento.
- Útil para agrupar contenido relacionado.


`<aside>`
- Contiene contenido tangencialmente relacionado con el contenido principal.
- Se usa para barras laterales, publicidad, etc.


`<footer>`
- Representa el pie de una sección o de la página.
- Típicamente contiene información de copyright, enlaces relacionados, etc.

```html
<footer>
  <p>&copy; 2024 Mi Sitio Web</p>
</footer>
```
`<figure>` y `<figcaption>`
- `<figure>` se usa para encapsular contenido multimedia.
- `<figcaption>` proporciona una descripción o leyenda para el contenido de `<figure>`.

```html
<figure>
  <img src="imagen.jpg" alt="Descripción de la imagen">
  <figcaption>Leyenda de la imagen</figcaption>
</figure>
```
Beneficios de usar etiquetas semánticas:

1.  Mejora la accesibilidad: Ayuda a las tecnologías de asistencia a interpretar el contenido.
2.  SEO optimizado: Los motores de búsqueda pueden entender mejor la estructura de la página.
3.  Código más legible: Hace que el código sea más fácil de entender y mantener.
4.  Mejora la experiencia del usuario: Facilita la navegación y comprensión del contenido.

Es importante usar estas etiquetas de manera apropiada y consistente para aprovechar al máximo sus beneficios. Recuerda que la semántica trata sobre el significado, no sobre la apariencia visual, la cual debe manejarse con CSS.

Ejemplo completo de una página HTML utilizando etiquetas semánticas:

```html
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Blog de Viajes</title>
</head>
<body>
    <!-- El header contiene el encabezado principal de la página -->
    <header>
        <h1>Mi Blog de Viajes</h1>
        <!-- Nav define la sección de navegación principal -->
        <nav>
            <ul>
                <li><a href="#inicio">Inicio</a></li>
                <li><a href="#destinos">Destinos</a></li>
                <li><a href="#sobre-mi">Sobre Mí</a></li>
                <li><a href="#contacto">Contacto</a></li>
            </ul>
        </nav>
    </header>

    <!-- Main engloba el contenido principal de la página -->
    <main>
        <!-- Section define una sección temática del documento -->
        <section id="ultimo-viaje">
            <h2>Mi Último Viaje</h2>
            <!-- Article representa un contenido independiente -->
            <article>
                <h3>Aventuras en los Alpes Suizos</h3>
                <!-- Figure se usa para contenido multimedia con su descripción -->
                <figure>
                    <img src="alpes.jpg" alt="Paisaje de los Alpes Suizos">
                    <figcaption>Vista panorámica de los Alpes Suizos</figcaption>
                </figure>
                <p>Mi reciente viaje a los Alpes Suizos fue una experiencia inolvidable...</p>
                <!-- Time se usa para fechas y horas -->
                <p>Fecha del viaje: <time datetime="2024-07-15">15 de julio de 2024</time></p>
            </article>
        </section>

        <!-- Otra sección temática -->
        <section id="proximos-destinos">
            <h2>Próximos Destinos</h2>
            <ul>
                <li>Tokio, Japón</li>
                <li>Machu Picchu, Perú</li>
                <li>Santorini, Grecia</li>
            </ul>
        </section>
    </main>

    <!-- Aside contiene información relacionada pero no esencial -->
    <aside>
        <h2>Recomendaciones de Viaje</h2>
        <ul>
```
